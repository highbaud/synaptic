# Team Knowledge Workflow — June 5

## Context
Marcus and Sarah both have large note backlogs that nobody can search. We agreed to pilot a shared PARA structure.

## Key points
- Marcus's current system: folder-per-project, no consistent naming, no frontmatter. Notes get lost after projects close.
- Sarah's current system: everything in one folder sorted by date. Fast to capture, impossible to retrieve.
- Agreed: the problem is not capture, it is retrieval. Both systems are organized for filing, not for finding.

## Decisions
Adopt PARA. Marcus sets up vault template. Sarah migrates her existing notes in Stage 3 (process the backlog) of the progressive reorganization plan.

## Actions
- [ ] Marcus: vault template by June 12
- [ ] Sarah: document current workflow for comparison
- [ ] Both: run first weekly connection surface on June 16
